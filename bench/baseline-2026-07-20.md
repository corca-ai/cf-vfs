# Local benchmark baseline — 2026-07-20

Environment: Apple M2, macOS 27.0, arm64, Node v24.6.0. Commands were
`npm run bench` and `npx vitest run test/storage-benchmark.test.ts`. Values are
one local warm-up followed by the median of three observations, not production
predictions. The executable check also applies a deliberately broad 10-second
per-scenario regression ceiling.

A zero observed delta means the operation completed between samples or garbage
collection lowered live memory; it does not mean zero allocation. The early
cancellation source exposed 1 MiB lazily and pulled only 16 KiB.

The workerd 1 MiB inline overwrite plus eager snapshot used four 256 KiB chunk
rows, reported 32 rows read and 16 rows written, and kept the already allocated
local SQLite database at 1,150,976 bytes. At the paid marginal row rates current
on the measurement date, those rows estimate to $0.000016032 beyond included
usage. The measured operation includes overwrite and snapshot but excludes the
diagnostic chunk-count query.
